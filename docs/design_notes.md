# Life Spending Designer Lite 設計メモ

## 1. このドキュメントの目的

このドキュメントは、Life Spending Designer Lite / タイ移住 月額生活費シミュレーターの計算ロジック、基準額、設計思想を記録するためのメモです。

画面文言ではなく、主に以下のような開発・保守上の判断を残します。

* 生活スタイル別プリセットの考え方
* 交通費の移動手段別調整ロジック
* 医療費自己負担の生活スタイル別ロジック
* 車・バイク維持費と交通費の分離
* 医療保険料の扱い
* 生活から試算する／予算から試算するの違い
* 予算配分モードの考え方
* レーダーチャートの考え方
* 結果コメントの考え方
* 家賃・部屋タイプ目安の考え方
* 兄弟アプリとの連携方針
* 実装上の注意点
* 今後の更新ルール

今後、計算ロジック、基準額、係数、生活スタイル別の前提を変更した場合は、必ずこのドキュメントも更新します。

---

## 2. アプリの役割

このアプリは、タイ移住後の月額生活費を試算するための生活設計ツールです。

主な目的は、タイで暮らす場合に、

* 月いくら必要か
* 生活費の中身はどうなるか
* 固定費と日常生活費のバランスはどうか
* 車・バイクや医療保険を入れると何が変わるか
* 生活スタイルによってどの費目が変わるか

を見える化することです。

また、兄弟アプリである「海外移住 資産寿命シミュレーター（簡易版）」に入力する、海外生活中の月額生活費の目安づくりにも使います。

### 「合計額を出す計算機」ではなく「暮らしの形を比較するアプリ」

このアプリは、単にタイ移住生活費の合計額を出すだけの計算機ではありません。

月額生活費を入力・比較することで、以下を見える化することが目的です。

* どのような生活スタイルになるか
* 車やバイクを持つと何が変わるか
* 医療保険を入れると月額費用がどれだけ上がるか
* 生活の満足度・安定性・健康・移動性・住環境のバランスがどう変わるか
* 「海外移住 資産寿命シミュレーター（簡易版）」に入力すべき「月額生活費」の目安を作ること

そのため、金額プリセットの位置づけは以下のとおりです。

```text
このアプリの金額プリセットは、実際の支出を1円単位で当てるためのものではなく、
生活スタイルごとの違いを比較し、退職後・移住後の生活費感覚をつかむための標準モデルである。
```

## 3. 生活スタイルの考え方

生活スタイルは、以下の4段階です。

```text
ミニマム生活
節約生活
標準生活
余裕生活
```

生活スタイルは、主に日常生活費の厚みを表します。

なお、4段階は単なる支出額の大小や贅沢度の違いではありません。

```text
生活スタイル4段階は、単なる支出額の大小や贅沢度ではなく、
生活の余白・選択肢・ストレス耐性の違いとして設計する。
```

対象となる主な費目は以下です。

```text
食費
日用品
交通費
外食費
交際費
趣味・レジャー
```

一方で、以下は生活スタイルとは別に扱います。

```text
家賃・管理費
光熱・通信費
医療保険
医療費自己負担
ビザ・滞在関連費
その他固定費
車・バイク維持費
```

生活スタイルの意味は以下の通りです。

### ミニマム生活

住まいも含めて支出をできるだけ抑え、外食・交際・趣味は最小限にする生活。

```text
ミニマム生活は「推奨生活」ではなく、生活の余地がかなり少ない下限モデルである。
短期的には成立しても、長期継続にはストレスが大きい可能性がある。
```

### 節約生活

基礎生活を保ちつつ、外食・交際・趣味は少なめに抑える生活。

### 標準生活

食費・外食・交際・趣味をある程度残す、無理の少ない生活。

```text
標準生活は、無理なく暮らすための基準モデルであり、アプリ内では比較の中心的な基準として扱う。
```

### 余裕生活

外食・交際・趣味・レジャーにも余白を持たせる生活。

## 4. 交通費の考え方

日常生活費側の「交通費」は、生活スタイルごとの基準交通費をもとにします。

ただし、車・バイクを持つ場合は、固定・準固定費側に「車・バイク維持費」を別途加算します。

そのため、車・バイクを持つ場合、日常生活費側の交通費は一部減額します。

これは、車・バイク維持費と交通費の二重計上感を避けるためです。

ただし、車やバイクを持っていても、以下のような支出は残るため、交通費はゼロにはしません。

* ソンテウ
* バイタク
* Boltなどの配車アプリ
* タクシー
* 駐車場
* 臨時移動
* 車やバイクを使わない短距離移動

### 移動手段別の交通費調整係数

```text
車・バイクなし：100%
バイクあり：70%
車あり：60%
車＋バイクあり：50%
```

車・バイクありの場合でも、日常生活費側の交通費は最低 1,000THB 残します。

### 例：ミニマム生活

基準交通費が 1,500THB の場合：

```text
なし：1,500THB
バイクあり：1,050THB
車あり：1,000THB
車＋バイクあり：1,000THB
```

ミニマム生活・バイクありの 1,050THB は、`1,500 × 0.7` の計算式を優先した結果です。
丸めて 1,000THB にするのではなく、係数計算を正とします。

### 例：標準生活

基準交通費が 2,100THB の場合：

```text
なし：2,100THB
バイクあり：1,470THB
車あり：1,260THB
車＋バイクあり：1,050THB
```

## 5. 車・バイク維持費の考え方

車・バイク維持費は、生活スタイルとは別に固定・準固定費側へ加算します。

車・バイク維持費には、以下のような費用を含めます。

* 燃料代
* 保険
* 税金
* 整備費
* 消耗品
* 通常の維持費

一方で、以下は含めません。

* 車両購入費
* ローン
* 減価償却
* 大きな修理費

交通費と車・バイク維持費は、以下のように分けて考えます。

```text
車・バイク維持費：所有することで毎月かかる固定的な費用
交通費：日常の補助的な移動費
```

### 維持費入力欄の表示制御

車・バイク維持費は、選択した移動手段に応じて加算します。誤解を避けるため、
選択されていない移動手段の入力欄は表示しません。

* 「なし」：詳細設定そのものを表示せず、車・バイク維持費は 0THB として扱う。
* 「バイクあり」：バイク維持費の入力欄だけを表示し、合計にはバイク維持費のみ加算。
* 「車あり」：車維持費の入力欄だけを表示し、合計には車維持費のみ加算。
* 「車＋バイクあり」：両方の入力欄を表示し、合計に両方を加算。

表示しないだけで session_state は壊さない。Streamlit はウィジェットが描画
されないと値を破棄するため、非ウィジェットのシャドーキーへ控えて再表示時に
復元し、ユーザーの手入力値を不要にリセットしない。生活スタイル別の初期値
ロジック（未編集なら新スタイルの推奨値に追随）も維持する。

## 6. 医療保険料の考え方

医療保険料は、移住開始時の年齢に応じて変わります。

このアプリでは、Pacific Cross の保険料表を参考にし、生活スタイルに応じて Premier / Premier Plus / Maxima の初期プランを切り替えた概算保険料を使っています（標準生活は Premier Plus を基準）。詳細は「15. 変更履歴メモ」を参照。

医療保険料はあくまで目安であり、実際の保険料は以下によって変わります。

* 年齢
* 健康状態
* 既往歴
* 補償内容
* 免責
* 通院特約
* 割引条件
* 保険会社の料率改定

医療保険料は固定・準固定費側に表示します。

## 7. 医療費自己負担の考え方

医療費自己負担は、毎月必ず発生する医療費ではありません。

以下のような費用を月割りした概算として扱います。

* 保険対象外の通院費
* 薬代
* 簡単な検査費
* 歯科・眼科などの自己負担
* 予防医療
* 人間ドック
* 軽い体調不良時の診察・薬代

生活スタイルによって病気リスクが変わるという意味ではありません。

生活スタイルによって、医療費への予備費をどこまで厚く見るかが変わる、という考え方です。

### 生活スタイル別の医療費自己負担

```text
ミニマム生活：1,500THB
節約生活：2,000THB
標準生活：2,000THB
余裕生活：3,000THB
```

### ミニマム生活

薬局での薬代、軽い体調不良時の最低限の通院・薬代を月割りで見る。
大きな検査、歯科治療、予防医療までは厚く見込まない。

### 節約生活

最低限より少し余裕を持ち、軽い通院、薬代、簡単な検査をある程度見込む。

### 標準生活

体調不良時の通院、薬代、簡単な検査、歯科・眼科などの一部自己負担を月割りで見る。

### 余裕生活

具合が悪いときに我慢せず病院へ行く、歯科、検査、人間ドック、保険対象外負担もある程度見込む。

## 8. 生活から試算する／予算から試算するの違い

このアプリには、以下の2つの試算方法があります。

```text
生活から試算する
予算から試算する
```

### 生活から試算する

これまでに選んだ条件を積み上げて、必要な月額生活費と生活バランスを見ます。

このモードでは、選択した生活スタイルに基づく生活費を満たす前提で、必要額を計算します。

主な出力は以下です。

```text
月額生活費（円）
月額生活費（THB）
生活バランス
固定・準固定費
日常生活費
```

### 予算から試算する

月額予算を自分で決めて、入力済みの固定費を反映しながら生活バランスを見ます。

このモードでは、まず固定・準固定費を優先し、その後、日常生活費に配分します。

予算が不足する場合は、選択した生活スタイルの基準額を満たせない費目が出ます。

予算が余る場合でも、無理に使い切らず、選択した生活スタイルの基準額まで配分し、余剰は予備費として残します。

### 固定費・準固定費と日常生活費の分離

```text
固定費・準固定費は、生活スタイルより先に確保すべき費用である。
予算配分モードでは、月額予算から固定費・準固定費を先に差し引き、残った金額を日常生活費に配分する。
```

理由は、家賃・管理費・通信費・ビザ費用・医療保険・車維持費などは、外食やレジャーよりも先に支払う必要があるためです。

### 低予算時の扱い

予算が足りない場合、固定費・準固定費を優先します。そのため、削られるのは主に日常生活費です。

極端な低予算ケースでは、以下の状態が起こり得ます。

```text
固定費すら満たせない
固定費は払えるが、日常生活費にほとんど回せない
食費や交通費は確保できるが、外食・交際・レジャー・趣味が削られる
標準生活には足りないが、節約生活なら成立する
```

0円や1万円などの入力は、実際に推奨する生活費ではなく、配分ロジックの耐久テストでもあります。

```text
低予算ケースは、実際に推奨する生活水準ではなく、配分ロジックの耐久テストでもある。
0円や1万円でもアプリが破綻せず、生活費不足を明確に示す必要がある。
```

0円時の「余り」表示についても、以下の考え方を持ちます。

```text
予算との差額が0円でも、それが「余裕」を意味するとは限らない。
特に予算自体が0円または極端に小さい場合は、緑色の余剰表示ではなく、
生活費不足・配分不能として扱う方が自然である。
```

### 高予算時の扱い

```text
月額予算が大きい場合でも、予算を無理に使い切らない。
選択した生活スタイルに必要な金額まで配分し、それ以上の余りは予備費として残す。
```

高予算時の余剰は、以下の意味を持ちます。

* 生活費を増やさなくても暮らせる余裕
* 突発費用への備え
* 為替やインフレへの余裕
* 「海外移住 資産寿命シミュレーター（簡易版）」側で資産寿命を延ばす余力

たとえば予算配分モードで月額予算50万円を入力した場合、標準生活の必要額だけが配分され、余剰が残ります。これは現仕様では正しい挙動です。標準生活の必要額を超えて、外食・レジャー・旅行などに自動配分しない設計だからです。

## 9. 予算配分モードの表記

予算から試算するモードでは、「日常生活費の配分」テーブルに以下の列を表示します。

```text
配分額(THB)
基準額(THB)
基準との差分(THB)
```

「基準額」は、ユーザーが個別に入力した希望額ではなく、選択した生活スタイルに基づいてアプリ側が持っている比較用の金額です。

そのため、「希望額」ではなく「基準額」と表記します。

## 10. レーダーチャートの考え方

このアプリでは、生活費の配分から、暮らしやすさやWell-Beingにつながる要素のバランスをレーダーチャートで表示します。

参考にしている考え方は、WHOQOL や OECD Better Life Index など、生活の質を複数領域で捉える考え方です。

ただし、指標自体はあくまで独自スコアです。

```text
レーダーチャートは、生活費配分の違いを直感的に比較するための独自スコアである。
学術的なQOL測定ではなく、生活費配分の比較用指標として扱う。
```

5つの指標は以下です。

```text
Life Satisfaction
Stability
Health
Mobility
Living Environment
```

### Life Satisfaction

外食・交際・趣味など、生活の楽しみ。

### Stability

固定費・医療保険・予備費など、生活の安定性。

### Health

食費・医療費・医療保険など、健康維持。

### Mobility

交通費・車・バイクなど、行動範囲。

### Living Environment

住居・光熱通信・日用品など、生活環境。

レーダーチャートは、選択した生活スタイルに対する充足度を示します。

予算が不足する場合、住居・医療・食費など生活の土台を優先し、そのうえで外食・交際・趣味などの余白をどこまで残せるかを確認します。

### すべて最大化することが目的ではない

```text
レーダーチャートはすべての指標を最大化することが目的ではない。
Life Satisfaction、Mobility、Living Environment、Health を高めるほど、生活費も増えやすい。
このアプリでは、どの指標を重視し、どの指標を抑えるかを考えるために使う。
```

## 11. 結果コメントの考え方

結果コメントは、以下の3つに分けて表示します。

```text
結果の分析
一言アドバイス
生活スタイルのメモ
```

3つの役割は混ぜすぎないようにします。

```text
コメントは、結果の分析、一言アドバイス、生活スタイルの説明を混ぜすぎない。
```

### 結果の分析

入力条件・計算結果から見た現在の状態を説明します。

固定費比率、移動手段、医療保険の有無などを反映します。

### 一言アドバイス

次に見直すならどこかを短く示します。

* 一言アドバイスは、選択中の生活スタイルで実行可能な助言にする。
* ミニマム生活では、これ以上生活スタイルを下げる助言（「一段控えめにする」など）は
  出さず、家賃・管理費・医療保険・光熱通信費などの固定費の見直しに誘導する。
* 余裕生活では「さらに余裕を持たせる」といった意味の薄い助言を出さず、標準・節約
  との差額確認に誘導する。
* 車を持っていない場合は、車を見直すとは言わない。車を持っている場合のみ、家賃や
  車の前提を見直すという表現を使う。
* 予算モードで予算と必要生活費がほぼ同じ（余りがわずか）のときは、「余った分は…」
  ではなく、予備費を別に確保できるか確認する助言にする。
* コメントは、選択条件と矛盾しないことを優先する。

#### 選択していない費目の見直しはすすめない

一言アドバイス・スコア連動コメントは、選択していない費目の見直しをすすめない。

* 医療保険なしの場合は、医療保険料の見直しではなく、医療費自己負担や万一の医療費への
  備え（余力）に言及する。ミニマム生活の一言アドバイスでも、保険なしのときは
  見直し候補に「医療保険」を含めない。
* 車・バイクなしの場合は、車・バイク維持費や車両前提ではなく、外出頻度や交通費の
  前提に言及する。
* これらは `has_insurance` / `vehicle_choice` を `advice_comment` と
  `score_low_comments` に渡して出し分ける。

### 生活スタイルのメモ

その生活スタイルで見落としやすい点・注意点を補足します。

* 一言アドバイスは、選択中の条件に対して次に見直す行動を示す。
* 生活スタイルのメモは、その生活スタイルで注意すべき点を補足する。
* 両者は同じ内容（「支出を抑えた前提」「バランスがよい」「快適さ重視」など）を
  繰り返さない。メモでは生活スタイルの再説明をせず、注意点に絞る。
* コメントは計算結果の信頼感に直結するため、条件との整合性と日本語の自然さを
  重視する（例：「ミニマム生活では」より「ミニマム生活は」が自然）。

### コメントの分岐に使う条件

コメント生成では、最低限以下の条件を見て分岐させると自然です。

```text
生活スタイル
医療保険あり・なし
車・バイクあり・なし
固定費比率
生活費充足率
予備費の有無
レーダーチャートの弱い指標
低予算かどうか
高予算で余剰が出ているか
```

分岐コメントの例（必要に応じて使用）：

```text
医療保険なし：
短期滞在や若年層なら成立する可能性はあるが、長期移住・高齢期では医療費リスクに注意。

車あり：
Mobility は上がるが、固定費が増える。予算に余裕がない場合はバイクや配車アプリ中心の生活も検討。

固定費比率が高い：
家賃・車・医療保険などで自由に使えるお金が圧迫されている可能性。

予備費が大きい：
予算を無理に使い切っていない。海外移住 資産寿命シミュレーター（簡易版）側では資産寿命を伸ばす余力として扱える。
```

## 13. 兄弟アプリとの連携

このアプリは、「海外移住 資産寿命シミュレーター（簡易版）」と連携して使うことを想定しています。

このアプリで月額生活費を試算し、その金額を「海外移住 資産寿命シミュレーター（簡易版）」に入力することで、資産寿命を試算しやすくします。

連携の流れは以下です。

```text
タイ移住 月額生活費シミュレーター
→ 月額生活費の目安を出す
→ 海外移住 資産寿命シミュレーター（簡易版）
→ 資産寿命を試算する
```

### 往復利用

兄弟アプリの基本的な使い方は、次の往復利用です。

```text
1. タイ移住 月額生活費シミュレーターで月額生活費を試算する
2. その金額を海外移住 資産寿命シミュレーター（簡易版）の海外生活中の月額生活費に入力する
3. 資産寿命を確認する
4. 資産寿命が短い場合は、タイ移住 月額生活費シミュレーターに戻って生活費、車・バイク、医療保険、家賃などを調整する
```

リンクはアプリ名に埋め込み、URLのベタ書きは画面上に出さない方針です。

## 14. 実装上の注意

Streamlit Cloud では、デプロイ反映のタイミングにより、`app.py` と `simulator.py` などの新旧状態が一時的にずれるように見えることがあります。

特に、関数引数を追加した場合、公開環境で古い関数定義が残っていると TypeError が出ることがあります。

そのため、関数引数を追加する場合は、できるだけ以下を守ります。

* デフォルト値を設定する
* 想定外の値は安全に扱う
* `None` や未知値でも落ちないようにする
* 後方互換的な呼び出しを検討する
* 公開環境でエラーが出た場合は、Streamlit Cloud の Reboot app とログ確認を行う

今回の交通費調整では、`vehicle_choice` の追加に関連して、後方互換的に処理するようにしました。

## 15. 変更履歴メモ

### 交通費ロジック

車・バイク維持費を固定費側に別計上しているため、日常生活費側の交通費を移動手段に応じて減額するロジックを追加しました。

### 医療費自己負担ロジック

医療費自己負担を一律ではなく、生活スタイルごとに変えるロジックにしました。
これは、生活スタイルによって病気リスクが変わるという意味ではなく、医療費への
予備費をどこまで厚く見るかの違いです。現行値は以下。

```text
ミニマム生活：1,500THB
節約生活：2,000THB
標準生活：2,000THB
余裕生活：3,000THB
```

### 生活スタイル別プリセットの現実寄り調整

公開アプリでは、ユーザーがまず「最低いくらで暮らせるのか」を見る可能性が
高いため、特にミニマム生活・節約生活を実例寄りに調整しました。

考え方：

* ミニマム生活は、実際に支出をかなり抑えて暮らしているタイ長期滞在者の支出感覚を
  参考にした。推奨生活ではなく、余白の少ない下限モデルである。
* ミニマム生活は、持ち家や安い住まい・自炊中心・バイク移動・外食/交際/趣味/物品購入を
  かなり抑える前提で組む。
* 標準生活・余裕生活は個人差が大きいため、現在の水準を大きく変えず、主に最小・
  控えめを現実寄りに調整した。

ユーザー入力できない日常生活費（食費・日用品・交通費・外食費・交際費・趣味）の
生活スタイル別プリセット（THB／月・合計）：

```text
ミニマム生活：6,000 / 2,000 / 1,500 / 2,500 / 1,500 / 2,000（合計 15,500）
節約生活：7,500 / 3,000 / 1,800 / 3,500 / 3,000 / 4,500（合計 23,300）
標準生活：9,000 / 4,000 / 2,100 / 5,000 / 6,000 / 9,000（合計 35,100）
余裕生活：12,000 / 5,000 / 4,000 / 8,000 / 10,000 / 11,000（合計 50,000）
```

### ユーザー入力できる固定費の初期値・推奨値

家賃・管理費・光熱通信費・医療保険料・医療費自己負担・ビザ・その他固定費・
バイク維持費・車維持費は、生活スタイルで強制固定せず、生活スタイル選択時の
初期値・推奨値として扱う。ユーザーが手入力で変えた値は、生活スタイル変更時も
保持する（現在値が前スタイルの推奨値のままのときだけ、新スタイルの推奨値に
追随する実装）。

* 家賃・管理費：賃貸/持ち家で大きく変わるため、生活スタイル別に決め打ちせず
  デフォルト 15,000THB のまま（ユーザー入力で調整）。
* 光熱・通信費（初期値）：最小2,800 / 控えめ3,500 / 標準4,500 / 余裕5,500。
  通信費は生活スタイルで極端に変わらないため差はつけすぎない。
* バイク維持費（初期値）：最小1,500 / 控えめ2,000 / 標準2,500 / 余裕3,000。
  所有・走行距離・新車中古などで決まり、生活スタイル差は小さめ。
* 車維持費（初期値）：最小6,000 / 控えめ7,000 / 標準8,500 / 余裕11,000。
  車格・任意保険・整備・遠出などで生活スタイル差が出やすい。
* その他固定費（初期値）：最小1,000 / 控えめ2,000 / 標準4,000 / 余裕5,000。
  日本側サブスク・保険・年会費などが残るため、ゼロ固定にはしない。

### 医療保険の生活スタイル別デフォルトプラン

医療保険は、生活スタイルごとに初期プランを変える。

```text
ミニマム生活：Premier
節約生活：Premier
標準生活：Premier Plus
余裕生活：Maxima
```

実装上は、Premier Plus 基準の月額概算にプラン係数（Premier≈0.69、
Premier Plus=1.0、Maxima≈1.16）を掛けて 100THB 単位に丸める。標準生活＝
Premier Plus は係数1.0で従来値と一致する。医療保険料は年齢・健康状態・補償内容・
免責・通院特約・割引条件などで変わるため、「自分で入力する」を選んだ場合は
ユーザー入力値を優先する。

#### 「自分で入力する」の入力値保持

ユーザーが「自分で入力する」で指定した医療保険料は、選択肢（入る／入らない／
自分で入力する）を切り替えても保持する。これは「ユーザーが手入力した値を不要に
リセットしない」という設計方針に合わせるため。Streamlit はウィジェットが描画されない
と session_state を破棄するため、車・バイク維持費と同じくシャドーキー
（`_persist_custom_insurance`）へ控え、再表示時に復元する。

* 自動計算の概算保険料（入る／年齢・スタイル・プラン連動）と、手入力保険料は
  役割を分ける。ユーザーが明示的に入力した値は、生活スタイル変更などで上書きしない。
* 「入らない」は従来どおり 0THB として扱う（手入力値そのものは保持される）。
* 初回表示時は従来どおり、年齢・スタイルに応じた自然な初期値を入れる。

### 生活スタイル名称の変更（表示名）

一般ユーザー向けの分かりやすさを高めるため、生活スタイル名を変更した。

```text
最小生活   → ミニマム生活
控えめ生活 → 節約生活
標準生活   → 標準生活（変更なし）
余裕生活   → 余裕生活（変更なし）
```

* 「最小生活」「控えめ生活」は計算モデルとしては分かるが、一般ユーザー向けの
  生活スタイル名としてはやや硬い。
* 「ミニマム生活」は、支出をかなり抑えた下限寄りの生活モデルとして表現する。
* 「節約生活」は、基礎生活を保ちながら日々の支出を意識して抑える生活として表現する。
* 名称変更は表示上の分かりやすさを高めるためのもので、金額ロジックは変更しない。
  内部キーも新名称へ統一し、各辞書・分岐を整合させた（計算結果は不変）。
